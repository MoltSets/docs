---
title: "IP to HEM"
api: 'POST /ip_to_hem'
description: "Resolve an IP address (with optional user agent) to hashed email addresses (MD5), ranked by confidence score."
---

## Connection

| | |
|---|---|
| **Method** | `POST` |
| **URL** | `https://api.moltsets.com/api/v1/tools/ip_to_hem` |
| **Auth** | `Authorization: Bearer YOUR_API_KEY` |
| **Content-Type** | `application/json` |

## Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `ip_address` | string | Yes | IPv4 address to resolve |
| `user_agent` | string | No | Browser user agent string — significantly improves match accuracy |
| `include_sha256` | boolean | No | When `true`, includes SHA-256 hashes alongside MD5 hashes in results |

<Tip>
  Always pass `user_agent` alongside `ip_address` when you have it. Capture both from your web server or analytics layer. Match accuracy improves substantially.
</Tip>

## Request

```bash
curl -X POST https://api.moltsets.com/api/v1/tools/ip_to_hem \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "ip_address": "8.8.8.8",
    "user_agent": "Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)",
    "include_sha256": true
  }'
```

## Response fields

| Field | Type | Description |
|---|---|---|
| `results` | array | List of HEM matches ranked by confidence |
| `results[].md5` | string | MD5 hashed email |
| `results[].sha256` | string | SHA-256 hashed email (only present when `include_sha256: true`) |
| `results[].score` | string | Confidence score between `0` and `1` — higher is more confident |
| `metadata.tokens_charged` | integer | Credits consumed by this request |
| `metadata.tokens_remaining` | integer | Credits remaining after this request |

<Info>
  Credits are only charged when results are found. If no HEMs are matched, `tokens_charged` will be `0` and `results` will be an empty array.
</Info>
