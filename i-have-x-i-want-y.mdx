## IP Resolution

IP resolution is best suited for B2B use cases — consumer ISP addresses will typically not resolve to a company. Passing a user agent string alongside the IP address significantly improves match accuracy for HEM lookups.

| I have… | I want… | Type in Claude… | Tool |
|---|---|---|---|
| IP address | Company associated with that IP | What company is behind IP 203.0.113.42 | `ip_to_company` |
| IP address | Hashed emails tied to that IP | Get hashed emails for IP 203.0.113.42 | `ip_to_hem` |
| IP address + user agent | Hashed emails with higher match accuracy | Get hashed emails for IP 203.0.113.42 with user agent [string] | `ip_to_hem` |
| IP address | Mobile advertising IDs (MAIDs) | Get mobile ad IDs for IP 203.0.113.42 | `ip_to_maid` |

## Common Workflows

Some tasks require chaining two tools together. These are the most useful combinations.

### Anonymous website visitor to identified contact

`ip_to_hem` → `hem_to_business_profile`

You have a website visitor IP address and want to know who it is. Resolve the IP to a hashed email, then resolve the HEM to a full business profile with name, title, and company.

"Resolve IP 203.0.113.42 to a hashed email, then get the business profile for that HEM."

---

### Anonymous website visitor to email address

`ip_to_hem` → `hem_to_email`

Same starting point as above, but you just want the plaintext email rather than the full profile.

"Resolve IP 203.0.113.42 to a hashed email, then get the plaintext email address."

---

### Anonymous website visitor to LinkedIn profile

`ip_to_hem` → `hem_to_best_linkedin`

Resolve an anonymous visitor to their LinkedIn profile. Useful if you want to review someone before reaching out.

"Resolve IP 203.0.113.42 to a hashed email, then find their LinkedIn profile."

---

### Business email to mobile phone

`hem_to_linkedin` → `enrich_phone`

You have a business email but no LinkedIn URL. Resolve it to a LinkedIn slug first, then use that to get a mobile phone number.

"Find the LinkedIn profile for john.smith@example.com, then get their mobile phone number."

---

### LinkedIn list to mobile ad audience

`linkedin_to_hashed_emails` → `hem_to_maid`

You have a list of LinkedIn profiles and want to push them into a mobile ad platform. Get all known HEMs for each profile, then resolve those to MAIDs.

"Get all hashed emails for linkedin.com/in/username, then resolve them to mobile ad IDs."

---

### CRM email list to ad retargeting audience

`business_email_to_personal_hem` → `hem_to_maid`

You have a CRM list of business emails and want to build a retargeting audience. Convert each business email to a personal HEM, then resolve to MAIDs.

"Convert john.smith@example.com to a personal hashed email, then resolve it to mobile ad IDs."
